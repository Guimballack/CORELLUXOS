# 05_UI_DESIGN_SYSTEM_SKILL

## Objective

This skill defines the global UI, visual consistency, design system and interface standardization rules for the ERP ecosystem.

The objective is to maintain:

* visual consistency
* reusable UI
* scalable interfaces
* predictable layouts
* enterprise-grade appearance
* AI-friendly UI architecture

The ERP must remain visually unified across all modules.

---

## Core UI Principles

Always prioritize:

* consistency
* readability
* usability
* scalability
* responsiveness
* accessibility
* reusable components
* predictable layouts

Never create inconsistent visual patterns.

---

## Design System Structure

The ERP must maintain a centralized design system.

Required structure:

```txt id="x9v2m6"
design-system/
├── COLORS.md
├── TYPOGRAPHY.md
├── SPACING.md
├── SHADOWS.md
├── BORDER_RADIUS.md
├── COMPONENT_PATTERNS.md
└── UI_RULES.md
```

All UI must follow centralized standards.

---

## Global UI Consistency Rules

Always standardize:

* buttons
* inputs
* modals
* cards
* tables
* badges
* dropdowns
* typography
* spacing
* shadows
* border radius

Never allow inconsistent UI between modules.

---

## Color System Rules

Colors must remain centralized.

Never use random hardcoded colors.

Always use variables.

Example:

```txt id="v5k8q1"
--color-primary
--color-secondary
--color-success
--color-warning
--color-danger
--color-background
--color-card
--text-primary
```

Maintain consistent semantic color usage.

---

## Typography Rules

Typography must remain consistent across the ERP.

Always standardize:

* font sizes
* font weights
* line heights
* heading hierarchy
* text spacing

Avoid random typography variations.

---

* a visually consistent ecosystem
