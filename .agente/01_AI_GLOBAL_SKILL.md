# 01_AI_GLOBAL_SKILL

## Objective

This file defines the global AI behavior, engineering governance, architecture standards, navigation principles and development rules for the entire ERP ecosystem.

All other skills must follow and respect this global skill.

The ERP must remain:

* modular
* scalable
* maintainable
* enterprise-ready
* AI-friendly
* predictable
* easy to navigate
* easy to maintain

---

## Global Architecture Principles

Always prioritize:

* clean architecture
* modularization
* separation of responsibilities
* low coupling
* maintainability
* reusable structures
* predictable organization
* enterprise scalability

Never prioritize fast temporary solutions over long-term architecture quality.

---

## Global Safety Rules

Never:

* alter existing functionality without explicit request
* break current workflows
* remove integrations
* change permissions silently
* modify business rules unexpectedly
* alter existing routes without authorization

Always preserve compatibility with:

* current modules
* current architecture
* current permissions
* current routes
* current design system

---

## Project Structure Rules

All project areas must be modularized.

Required structure:

```txt
AREA/
├── pages/
├── components/
├── modals/
├── forms/
├── tables/
├── services/
├── state/
├── events/
├── validations/
├── constants/
├── types/
└── styles/
```

Never mix unrelated responsibilities inside the same structure.

---

## Global Naming Convention

Always use predictable enterprise naming.

Pattern:

```txt
MODULE_FUNCTION_DETAIL
```

Examples:

```txt
SCREEN_STOCK_ENTRY
BTN_SAVE_EMPLOYEE
TABLE_PRODUCTS
MODAL_EDIT_USER
INPUT_EMPLOYEE_NAME
SERVICE_AUTH_LOGIN
```

Never use:

* button1
* modal2
* temp
* test
* newFile
* random names

---

## Navigation Principles

The entire project must be optimized for fast AI navigation.

Every structure must allow instant identification of:

* modules
* screens
* components
* routes
* permissions
* services
* events
* styles
* states

Architecture must reduce unnecessary project-wide searching.

---

## Module Isolation Rules

Each module must behave as an isolated ecosystem.

Modules may communicate only through:

* services
* shared interfaces
* centralized state
* approved integrations

Avoid direct coupling between modules.

---

## Global Design System Rules

Maintain visual consistency across the ERP.

Always standardize:

* buttons
* forms
* tables
* modals
* spacing
* typography
* colors
* shadows
* border radius

Never create inconsistent UI patterns.

---

## Clean Code Principles

Always:

* reuse existing structures
* simplify logic
* isolate responsibilities
* centralize constants
* organize imports
* remove dead code
* reduce duplication

Never:

* duplicate business logic
* create giant files
* create hidden dependencies
* mix UI and business logic

---

## Performance Principles

Always prioritize:

* optimized rendering
* reusable services
* modular components
* scalable state management
* minimal re-rendering

Avoid:

* duplicated listeners
* repeated calculations
* oversized components
* unnecessary queries

---

## Security Principles

Always protect:

* authentication
* permissions
* protected routes
* user data
* business rules

Never expose sensitive internal logic unnecessarily.

---

## Scalability Principles

* future modules

* future scalability
