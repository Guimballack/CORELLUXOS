# 10_DOCUMENTATION_SKILL

## Objective

This skill defines the global documentation, indexing, structural mapping and synchronization standards for the ERP ecosystem.

The objective is to maintain:

* self-documenting architecture
* synchronized project maps
* predictable documentation
* scalable indexing
* AI-friendly navigation
* enterprise-grade project organization

Documentation must remain continuously updated alongside ERP evolution.

---

## Core Documentation Principles

Always prioritize:

* clarity
* synchronization
* predictability
* traceability
* maintainability
* navigation optimization

Documentation must reflect the real project structure at all times.

---

## Documentation Structure Rules

All documentation must remain organized by responsibility.

Required structure:

```txt id="d8m2q5"
docs/
├── architecture/
├── design-system/
├── skills/
├── standards/
├── workflows/
└── ai-index/
```

Avoid chaotic or duplicated documentation structures.
